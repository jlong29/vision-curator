# Architecture

`vision-curator` sits between edge package production and model training.

- `thermal-data-engine` produces immutable Phase 2 clip packages.
- `vision-curator` validates, indexes, scores, queues, and releases curated datasets.
- `vision-trainer` consumes immutable dataset releases.

CVAT and FiftyOne are optional integrations. Core validation, scoring, queue generation, and release building must run without them.

## Current System Direction

The v2 workspace design keeps responsibilities split:

- the edge node generates detector and tracker signal,
- `vision-curator` decides which signal is usable data,
- `vision-trainer` trains and evaluates candidate models from curated releases.

Phase 2 clip packages are the primary input to this repo. Phase 1 YOLO packages remain useful for bootstrap smoke tests and audit compatibility, but they are not the authoritative curation boundary.

## EgoHumans Calibration Readiness

The next experiment uses EgoHumans Lego Assembly as a controlled calibration dataset. The edge node has already produced package outputs as if EgoHumans were unlabeled deployment video. `vision-curator` must therefore keep three label namespaces separate:

- hidden oracle labels imported from EgoHumans ground truth,
- revealed gold labels selected through review or seed policies,
- teacher pseudo labels generated by edge inference and trust scoring.

Only teacher pseudo labels may influence trust scoring. Hidden oracle labels are for evaluation and simulated reveal workflows, not ordinary training labels.
